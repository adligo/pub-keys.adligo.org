# Jenkins 2026-02-28

We usually prefer the war because we can run it directly or create our own 
Docker image and include it.  When we run it on bare metal we have better control 
over usage of services inside Docker and Web Browsers on Windows, Mac and Linux.

- [https://www.jenkins.io/doc/book/installing/war-file/](https://www.jenkins.io/doc/book/installing/war-file/)
- [https://www.jenkins.io/download/](https://www.jenkins.io/download/)


This uses a environment variable FABRICATE_APPS_DIR which is usually /apps on Unix (Linux,Max) or /c/apps on Windows, but is configurable.

```
cd $FABRICATE_APPS_DIR

mkdir jenkins
cd jenkins
wget https://get.jenkins.io/war/2.552/jenkins.war
sha256sum jenkins.war > jenkins.war.sha256
cat jenkins.war.sha256
diff jenkins.war.sha256 ~/pub-keys.adligo.org/pub-keys/io/jenkins/download/jenkins.2.552.war.sha256

sha512sum jenkins.war > jenkins.war.sha512
diff jenkins.war.sha512 ~/pub-keys.adligo.org/pub-keys/io/jenkins/download/jenkins.2.552.war.sha512
```

Or

```
cd $FABRICATE_APPS_DIR
```
